# Routing

This document defines operating routes for planning, execution, and validation.

## Planning phase
Read and maintain:
- `project/vision/*`
- `project/docs/features/*`
- `project/docs/task_groups/*`
- `project/docs/tasks/*`
- `project/docs/priorities/*`

Planning updates must be written to repository files. Planning is not complete if it remains only in chat.

## Execution phase
Read:
- `project/now/description.md`
- `project/now/prompt.md`

Execution must stay within prompt scope and rule boundaries.

## Validation phase
Check:
- `project/evidence/`
- forbidden mutations against protected and restricted paths

Validation is complete only when evidence exists and file boundary rules were respected.
